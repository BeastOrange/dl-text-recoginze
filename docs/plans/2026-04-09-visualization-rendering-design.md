# 可视化渲染统一与重叠修复设计

## 目标

统一端到端可视化输出风格，解决当前预览图中标签重叠、文本过长、样式粗糙的问题，使结果图更适合客户交付与演示。

## 当前问题

- 标签直接画在框左上角，密集框场景容易重叠
- 没有标签背景，复杂背景下可读性差
- 图内同时承载完整文本与分类，信息密度过高
- 分离式端到端与统一多任务端到端各自维护一套绘制逻辑，样式不一致

## 设计决策

采用“统一渲染器 + 图内短标签 + 右侧图例栏”的方案：

1. 抽取共享渲染模块，统一负责框线、标签、图例绘制。
2. 图内只显示短标签：
   - 优先为 `#序号 + 短文本`
   - 文本过长时截断
   - 小框降级为仅显示编号
3. 右侧图例栏展示完整信息：
   - 编号
   - 识别文本
   - 识别置信度
   - 分类标签
   - second-pass 标记
4. 标签位置采用简单避让：
   - 优先上方
   - 依次尝试下、右、左
   - 避免与已有标签框重叠
5. 统一渲染器同时服务于：
   - `src/dltr/pipeline/end_to_end.py`
   - `src/dltr/models/end2end_system.py`

## 非目标

- 不引入复杂自动排版库
- 不修改推理结果 JSON 结构
- 不重做 Streamlit 页面布局，只提升底层预览图质量

## 验证策略

- 单元测试验证预览图正常生成
- 验证多框场景仍可输出 PNG
- 验证长文本会被截断或降级
- 验证统一会话和分离式会话都走同一渲染逻辑
