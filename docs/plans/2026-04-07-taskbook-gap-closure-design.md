# Taskbook Gap Closure Design

## Goal

补齐任务书审查中确认的四个主要缺口：

- 统一多任务协同训练与统一端到端系统
- 公开数据主线实验结果落盘
- 真实 ONNX 导出
- 曲线/非四点文本几何支持增强

## Recommended Approach

采用“统一多任务模型 + 兼容现有分离式链路”的方案：

1. 新增统一多任务模型，使用共享视觉主干，分别连接检测头与识别头。
2. `train end2end` 改为训练统一模型，并输出统一 checkpoint 与联合训练摘要。
3. `evaluate end2end` 支持统一 checkpoint 推理，同时保留现有检测/识别双 checkpoint 推理。
4. 识别主线改为优先使用检测标注裁切得到的公开场景文本行样本。
5. 检测/裁切/评估链路从固定四点框放宽到通用偶数点 polygon，并在需要透视矫正时回退到最小外接矩形。
6. `export onnx` 从“生成计划”改为真实导出，并在缺依赖时给出明确错误。

## Tradeoffs

- 不强求旧检测模型与新统一模型共享完全相同的网络定义，避免高风险大重构。
- 保留现有分离式训练命令与推理路径，降低兼容性风险。
- 主线实验先支持受控步数/批次数限制，保证当前设备可落盘真实结果；正式全量训练仍保留配置化入口。

## Validation

- 单元测试覆盖：
  - 统一多任务训练摘要输出
  - 统一 checkpoint 推理
  - polygon 泛化裁切/绘制/IoU
  - 真实 ONNX 导出命令
- 定向命令验证：
  - 数据裁切
  - 统一训练
  - 统一推理
  - 报告汇总
